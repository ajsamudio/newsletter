# NewsLetter

A personal weekly brief on **tech, AI, global economics, US stocks, world & general news, sports & prediction markets** (NBA / NFL / Kalshi / Polymarket), plus a closing **community Vibes** section — generated by a Claude routine, archived as markdown, emailed to me.

See [PLAN.md](PLAN.md) for the full design rationale.

## Layout

| File | What it is |
|------|------------|
| `newsletter-prompt.md` | The master prompt the routine runs every time. Edit this to change content, tone, or sources. |
| `source-audit-prompt.md` | Quarterly prompt that proposes additions/removals to the source lists — keeps the brief from calcifying around fading sources. |
| `template.md` | Reference for the output structure. The prompt points here. |
| `archive/` | One markdown file per issue, named `YYYY-MM-DD.md`. |
| `PLAN.md` | Full plan, phases, verification steps. |

## How to run manually (any time)

Open this folder in Claude Code and say:

> Run the newsletter generation as defined in `newsletter-prompt.md`.

Claude will WebSearch + WebFetch the four sections, write the issue to `archive/{today}.md`, and print a one-line summary.

## How the routine works (Phase 2 onward)

The routine is registered via the `scheduled-tasks` MCP. It runs on Anthropic's infrastructure on a cron schedule — your laptop doesn't need to be on.

- **Current schedule (test phase):** daily at 8:07 am local.
- **Target schedule (after test phase):** Thursdays at 8:07 am local.

To list / pause / change the routine, ask Claude:

> List my scheduled tasks.
> Pause the newsletter routine.
> Change the newsletter routine to run weekly on Thursday at 7am.

## Editing the newsletter

Almost all changes are made by editing `newsletter-prompt.md`:

- Want different sources? Edit the "Preferred sources" lines.
- Want longer / shorter stories? Edit the "3–5 stories" counts or tone rules.
- Want a new section? Add it under section 2, including source guidance.

The routine reads the file fresh every run, so changes take effect on the next fire.

## Email delivery (Phase 3)

Sent via [Resend](https://resend.com) free tier.

1. Sign up at resend.com using `antsamudio99@gmail.com`.
2. Create an API key, save it as `RESEND_API_KEY` in the routine secrets.
3. The prompt's Section 6 step 2 already has the POST template — uncomment / enable it once the key is in place.

Free tier note: without a verified domain, Resend only delivers to the address you signed up with. That's why you must sign up using the recipient address.

## Troubleshooting

- **Empty section in an issue:** WebSearch found nothing usable. Often a weekend / holiday — usually self-resolves next run.
- **Same story two days in a row:** rule #5 in `newsletter-prompt.md` should prevent this — tighten the de-dup language if it slips.
- **Routine didn't fire:** ask Claude `List my scheduled tasks` to confirm it's registered and the last-fired timestamp.
- **Email didn't arrive:** check spam, then verify the Resend dashboard shows a delivered event for the timestamp.
